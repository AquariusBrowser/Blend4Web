安装顺序¶

安装Blender¶

直接在 Blender 中创作 3D 场景，这是开放源码软件和免费分发软件。

应使用当前的稳定版本的Blender。它可以从 官方网站 下载 。

解压缩文件归档¶

稳定版作为压缩包发布 (blend4web_sdk_free_YY_MM.zip – 免费 SDK, blend4web_sdk_pro_YY_MM.zip – 商业 SDK). 只要在某处解开这个压缩包。

SDK 安装顺序¶

  1. 打开Blender。

  2. 打开用户首选项面板 File > User Preferences....

  3. 打开 File 选项卡。

  4. 在 Scripts 中设置 SDK 目录的路径。

  5. 单击 Save User Settings 按钮。

  6. 重新启动Blender。

验证是否工作:

在 File > Export 菜单 Blend4Web (.json) 和 Blend4Web (.html) 选项应该出现。

注解

如果您打算使用只用法线编辑器，你可以考虑安装 Blend4Web 插件 而不是 SDK。

插件只能导出场景到 HTML格式 和有限功能 (例如，它没有 项目管理，示例场景，用户手册，等等)，但它仍然拥有一切让你可以创建一个简单的应用程序。

它可以和其他 Blender插件 的安装方式相同。


Blend4Web模式¶

为了查看该引擎设置，从顶部面板菜单中选择Blend4Web：

本地开发服务器¶

设置本地开发服务器可以在 File > User Preferences... (热键 Ctrl-Alt-U)中找到。 在这里，您可以更改端口号被用于运行开发服务器（6687默认情况下），也使其在Blender启动时启动。要做到这一点，能够有 Run on Startup 的附加 ​​设置选项。


改变了的本地开发服务器设置后需要重新启动Blender。

它有可能为本地服务器来处理外部请求。要做到这一点启用该选项 Enable External Requests.

如果你选择不自动启动服务器，您可以随时手动做到这一点：转到 渲染 选项卡，并按 开发服务器 面板上的 启动服务器 按钮 ︰


注解

如果不指定 Blend4Web SDK 的路径，不能启动本地开发服务器。在这种情况下会显示相应的消息，覆盖到 启动服务器 按钮。

如果服务器无法运行，会显示一条错误消息来说明原因：

如果已经被一些其他的应用程序的占用掉了服务器端口，可能会出现这样的错误。

按 SDK Index 按钮在浏览器中打开 Blend4Web SDK 的索引 web 页。此页面在 http://localhost:6687 。


结果是，在您的操作系统的默认浏览器将启动。

项目管理 按钮可用于打开 项目管理.

Fast Preview 按钮加载当前在Blender当中打开的场景到 viewer 中。

按下 停止服务器 按钮可以停止服务器。Blender关闭时，它也会停止。

运行浏览器和演示¶

索引页面中包含指向发动现场观众和演示应用。一个 WebGL-capable browser 才能运行这些应用程序。


注解

如果不能正确显示的SDK应用程序，或错误消息显示，按照 启动时的问题 中的说明部分。

更新¶

更新 SDK¶

在更新之前，我们建议您使用 project export 工具备份的所有开发中的项目.

有两种方法来更新 SDK。

首先，可以完全删除 SDK 文件夹中，在那之后，提取新版本文件夹并将路径设置为它在Blender设置，就像你第一次安装 SDK一样。

第二，你可以简单地使用新的覆盖过时的 SDK 文件。此方法被认为是不太 “纯净”，但在大多数情况下，它不应导致任何问题。

完成更新后，您可以使用 项目导入 工具导入所有已保存的项目。

导出和查看场景¶

创建的场景可以以 HTML 格式导出。使用 File > Export > Blend4Web (.html) 或者 File > Export > Blend4Web (.json) 菜单选项，然后选择导出文件的路径。场景导出完全描述在 自己章节. 可以用任何支持 WebGL的浏览器打开生成的 HTML 文件。
